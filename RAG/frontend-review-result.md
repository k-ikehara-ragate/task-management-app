# フロントエンド実装レビュー結果

**対象:**  

- `frontend/app/pages`  
- `frontend/app/components`  
- `frontend/nuxt.config.ts`

**参照観点:** `/home/shimazu/work/RAG/revew.md` の「画面遷移の整合性」「Nuxt 4 の構成」「UIの実装」に基づく確認。

---

## 1. 画面遷移の整合性

### できている点

- **トップ → 一覧:** トップページの「タスク一覧へ」「一覧・編集」から `/tasks` への遷移あり。
- **トップ → 新規作成:** クイックアクション「新規タスク登録」から `/tasks/new` への遷移あり。
- **新規作成 → 一覧:** 登録後に `router.push('/tasks')` で一覧へ戻る実装あり。キャンセルは NuxtLink で `/tasks`。

### 懸念・不足

- **編集画面がない:**  
`tasks/[id].vue` や `tasks/[id]/edit.vue` など編集用ページが存在しない。  
RAG の「トップ → 一覧 → 新規作成 → **編集**」の「編集」が未実装。
- **一覧からの編集導線が未実装:**  
`tasks/index.vue` はプレースホルダーのみで、一覧表示・担当者フィルタ・期日順・編集リンクがまだない。  
このため「一覧 → 編集」の流れが作れない状態。

---

## 2. Nuxt 4 の構成

### できている点

- **ディレクトリ構造:** Nuxt 4 のデフォルト `srcDir: 'app'` に合わせ、`app/pages`・`app/components`・`app/layouts`・`app/app.vue` が適切に配置されている。
- **ページ定義:** `app/pages/index.vue`（トップ）、`app/pages/tasks/index.vue`（一覧）、`app/pages/tasks/new.vue`（新規）がファイルベースルーティングとして成立。
- **レイアウト:** `app/layouts/default.vue` でサイドバー＋メインの土台があり、`app.vue` で `NuxtLayout`・`NuxtPage` を利用。テーマは `useTheme()` で app ルートに適用。

### 懸念・考慮不足

- **nuxt.config.ts:**  
`compatibilityDate`・`devtools`・`css`・`devServer.port` のみで最小限。  
現状で動くが、以下は今後の拡張時に検討するとよい。
  - `app/` を明示したい場合の `srcDir: 'app'`（Nuxt 4 デフォルトが `app` のため必須ではない）。
  - ルートやエイリアス（`~/` が `app/` を指すか）の明文化が必要になった場合の設定。
- **編集ルートの追加:**  
編集画面を実装する場合、`app/pages/tasks/[id].vue` または `app/pages/tasks/[id]/edit.vue` の追加が Nuxt 4 の規約に沿った形となる。

---

## 3. UI の実装

### できている点

- **トップページ:** ヘッダー・StatCard・ウェルカム・クイックアクションのレイアウトが CSS 変数（tokens）で統一され、崩れは見当たらない。
- **新規作成フォーム:** タイトル・担当者・期日の入力とラベル、登録／キャンセルボタンの配置が整理されている。`required` により未入力時は送信不可。
- **コンポーネント:** `AppDashboardHeader`・`AppDashboardSidebar`・`AppStatCard`・`AppThemeToggle` が共通化され、役割が分かれている。
- **アクセシビリティ:** ラベルと `id` の対応、`aria-label`／`aria-hidden` の使用があり、フォームとテーマ切替で配慮されている。

### 懸念・考慮不足

- **バリデーション・エラー表示:**
  - クライアント側の詳細バリデーション（タイトル文字数制限、期日の過去日禁止など）は未実装。
  - エラー時のメッセージ表示（例: 「タイトルは必須です」「期日は今日以降を指定してください」）がなく、HTML5 の `required` とブラウザ標準の表示に依存している。
  - 登録 API や use case 層でエラーが返ってきた場合の表示も未実装。
- **一覧・編集の UI:**  
一覧は「次のフェーズで実装」のプレースホルダーのみ。編集画面がないため、編集用フォームやステータス変更などの UI は未実装。
- **サイドバーの導線:**  
サイドバーには「ダッシュボード」「タスク一覧」のみ。「新規タスク登録」はトップのクイックアクションにはあるが、サイドバーにはない。一覧画面から新規作成へすぐ行けるリンクも、一覧未実装のためなし。
- **新規作成後の体験:**  
登録後は `localStorage` に保存し `/tasks` に遷移するが、一覧が未実装のため登録したタスクが画面に反映されない。次のフェーズで一覧を実装する前提であれば、その旨をプレースホルダーやコメントで明示しておくとよい。

---

## 4. その他（実装品質・型・データ）

- **型定義:** `~/types/task.ts` で `Task`・`TaskFormInput`・`TaskStatus` が定義され、`tasks/new.vue` で利用されている。`any` に頼っていない。
- **Composables:** `useAssignees()`・`useTheme()` が適切に切り出されている。
- **新規作成の保存先:** 現状は `localStorage` のみ。バックエンドや API 連携を入れる場合は、composable や API 層で差し替えしやすい構造になっている。

---

## 5. まとめ（実装していく上で問題ないか）


| 観点         | 判定       | 備考                                  |
| ---------- | -------- | ----------------------------------- |
| 画面遷移の整合性   | 要対応      | 編集画面および一覧→編集の導線が未実装。トップ↔一覧↔新規は問題なし。 |
| Nuxt 4 の構成 | 問題なし     | `app/` 以下の構成は Nuxt 4 の規約に沿っている。     |
| UI（レイアウト）  | おおむね問題なし | 既存画面のレイアウトは崩れていない。一覧・編集は未実装。        |
| バリデーション等   | 要検討      | 詳細バリデーションとエラー表示の追加を検討するとよい。         |


**結論:**  
現時点の「トップ・一覧（プレースホルダー）・新規作成」と Nuxt 4 構成・既存 UI は、実装を進める土台として問題ない。  
一方で、RAG で期待されている「編集」までの画面遷移と、一覧・編集の UI・バリデーション・エラー表示は、今後のフェーズで実装・強化する必要がある。
